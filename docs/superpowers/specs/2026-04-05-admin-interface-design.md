# Design: Админ-интерфейс для управления слотами

## Overview

Добавление админ-интерфейса для управления доступными слотами. Используется существующая система правил доступности (availability rules). Для MVP пользователь и админ — один человек.

## UI Архитектура

### Переключение режимов

В правом верхнем углу страницы — кнопка-иконка (шестерёнка ⚙️). По клику — dropdown-menu:
- "Режим записи" (активен по умолчанию)
- "Админ-режим"

Выбор режима сохраняется в localStorage. Переключение мгновенное, без перезагрузки страницы.

### Режим записи (существующий)

- DatePickerInput для выбора даты
- SlotPicker с слотами по 30 минут
- BookingForm для оформления записи
- BookingList для просмотра предстоящих встреч (для владельца)

### Админ-режим

Страница с двумя секциями:

**Секция А: Правила доступности**
- Заголовок: "Правила доступности"
- Список существующих правил с отображением:
  - Для recurring: "Каждый [день(и)] с [время] до [время]"
  - Для one-time: "[дата]: с [время] до [время]"
- Кнопка "Удалить" для каждого правила
- Кнопка "Добавить правило" → открывает модальное окно

**Секция Б: Заблокированные дни**
- Заголовок: "Заблокированные дни"
- Список заблокированных дат
- Кнопка "Удалить" для каждой даты
- Кнопка "Заблокировать день" → DatePicker для выбора даты

## Форма создания правила (модальное окно)

```
Заголовок: "Новое правило доступности"

Дни недели: [Пн] [Вт] [Ср] [Чт] [Пт] [Сб] [Вс]

Время:
  С: [ввод времени]  До: [ввод времени]

[Отмена] [Сохранить]
```

Логика сохранения: при нажатии "Сохранить" создаётся по одному recurring правилу для каждого выбранного дня недели. Каждое правило содержит один временной диапазон.

## API Использование

| Операция           | Метод  | Путь                        |
|--------------------|--------|-----------------------------|
| Список правил       | GET    | /api/availability-rules     |
| Создание правила    | POST   | /api/availability-rules     |
| Удаление правила    | DELETE | /api/availability-rules/{id}|
| Список blocked days| GET    | /api/blocked-days           |
| Блокировка дня      | POST   | /api/blocked-days           |
| Разблокировка дня   | DELETE | /api/blocked-days/{id}      |

## Компоненты

### Новые компоненты

1. **ModeToggle** — кнопка-иконка с dropdown для переключения режимов
2. **AdminPage** — страница админ-режима
3. **AvailabilityRuleList** — список правил с удалением
4. **AvailabilityRuleForm** — форма создания правила (модалка)
5. **BlockedDayList** — список заблокированных дней

### Модифицируемые компоненты

1. **Layout** — добавить ModeToggle в header

## Состояние (State)

Режим хранится в localStorage под ключом `booking-mode`:
- `"client"` — режим записи (по умолчанию)
- `"admin"` — админ-режим

## Визуальные требования

- Используется Mantine UI (существующий стек)
- Русский язык для всех подписей
- Кнопка-иконка в правом верхнем углу
- Список правил — Card-компоненты или простой список
- Форма в модальном окне Modal из Mantine

## Тестирование

- Проверка переключения режимов
- Проверка создания правила (выбор нескольких дней)
- Проверка удаления правила
- Проверка блокировки/разблокировки дня
- Проверка сохранения режима в localStorage
